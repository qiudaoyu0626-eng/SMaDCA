一个用于个人现金流、应急储备、基金/ETF 持仓、定投计划与资产再平衡分析的 Codex Skill。

本 Skill 的目标不是预测市场短期涨跌，也不是给出高频交易建议，而是帮助用户建立一套可长期执行的个人财富管理流程：先保证现金流和应急金安全，再根据风险承受能力进行分散配置、持续定投和定期再平衡。

功能概述

本 Skill 可以帮助用户完成以下任务：

分析月收入、必要支出、弹性支出和债务情况
计算月结余和储蓄率
评估应急金覆盖月数和应急金缺口
分析基金、ETF、黄金等持仓占比
判断单一资产、行业、国家或主题是否过度集中
对比当前持仓比例和目标配置比例
计算组合在不同下跌幅度下的潜在亏损金额
给出定投优先级和再平衡建议
生成发薪日执行计划
记录投资决策前提，例如持有周期、可接受回撤和调整条件

输出内容

脚本会输出三类结果：

1. Cash Flow

现金流分析包括：

月结余
储蓄率
应急金覆盖月数
应急金目标金额
应急金缺口

计算公式：

月结余 = 月税后收入 - 必要支出 - 弹性预算 - 债务还款
储蓄率 = 月结余 / 月税后收入
应急金覆盖月数 = 当前应急金 / 每月必要支出
应急金目标 = 每月必要支出 × 目标覆盖月数
应急金缺口 = max(0, 应急金目标 - 当前应急金)
2. Portfolio

投资组合分析包括：

组合总金额
每个持仓的当前金额
每个持仓的当前占比
每个持仓相对目标比例的偏离
20%、30%、40% 下跌压力测试

计算公式：

资产占比 = 单项资产金额 / 长期投资组合总金额
目标偏离 = 当前占比 - 目标占比
压力测试亏损 = 组合总金额 × 下跌幅度
3. Warnings

风险提示包括：

应急金不足
月现金流为负
单一资产占比过高
单一持仓超过 30% 时提示集中风险
单一持仓达到或超过 40% 时提示高度集中风险
Codex 使用流程

Codex 使用本 Skill 时，推荐按以下流程执行：

识别用户是否在询问现金流、存钱、基金持仓、定投、资产配置或再平衡。
读取 SKILL.md，确认分析流程和输出要求。
如果用户没有提供完整数据，优先提取已有信息，只补问会 materially 改变结论的关键问题。
读取 references/default-profile.md，作为当前用户的默认参考，但不要把它当作永久事实。
读取 references/methodology.md，使用其中的计算公式、风险分类和决策优先级。
如果用户提供了结构化数据，则调用 scripts/analyze_portfolio.py 进行计算。
根据脚本输出结果，结合方法论生成中文分析。
输出最终建议时，先给结论，再给现金流、应急金、持仓风险、目标配置、定投计划、再平衡规则和缺失信息。
分析原则

本 Skill 遵循以下原则：

1. 先安全，再收益

在做长期投资之前，优先确认：

是否有高息债务
应急金是否足够
未来三年内是否有大额用钱需求
当前现金流是否稳定

如果存在高息债务，应优先偿还债务。
如果应急金不足，应优先补足应急金。
如果短期内需要用钱，不应把这部分资金投入高波动资产。

2. 不预测短期市场

本 Skill 不做以下事情：

不预测短期涨跌
不承诺收益
不鼓励频繁交易
不建议借钱投资
不默认推荐杠杆
不把历史收益当作未来收益保证
3. 重视集中度风险

如果单一窄基、行业、国家、主题或个股占比过高，即使它内部持有很多证券，也不能简单认为它是充分分散的。

风险判断规则：

单一窄基、行业、主题或国家暴露超过 30%：提示集中风险
达到或超过 40%：提示高度集中风险
如果组合虽然资产数量多，但底层重仓高度重叠，也应提示重叠风险
4. 优先用新增资金再平衡

除非存在严重集中、产品缺陷、高费用、流动性问题或目标比例严重偏离，否则优先建议通过新增定投资金进行再平衡，而不是立即卖出已有资产。

例如：

如果 Nasdaq 100 当前占比明显高于目标比例，可以暂停或降低 Nasdaq 100 的新增定投，把新资金优先投入低配资产，逐步把组合拉回目标比例。
5. 用人民币金额解释风险

不要只告诉用户“可能下跌 30%”。
应同时说明对应的人民币亏损金额，例如：

如果当前长期投资组合为 100,000 元，下跌 30% 对应账面亏损约 30,000 元。

这样用户更容易判断自己是否真的能承受波动。

注意事项

本 Skill 只用于个人财务规划和投资组合分析，不构成确定性收益承诺。
所有收益测算都应视为情景分析，而不是市场预测。
涉及具体基金产品、费用、税务、交易限制和监管规则时，应使用权威来源核对最新信息。
用户在执行任何重大投资调整前，应确认自己的现金流、债务、应急金、用钱期限和最大可承受回撤。
